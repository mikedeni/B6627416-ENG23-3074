# 🚀 K8S Infrastructure MVP: Nginx + Postgres + Jenkins + Monitoring

A production-ready Kubernetes infrastructure boilerplate running on **Kind** (Kubernetes in Docker). This project includes a web server, database, automated CI/CD pipeline, and full observability.

---

## 🏗️ System Architecture

- **Frontend/Proxy**: Nginx (3 Replicas) with Ingress Controller.
- **Database**: PostgreSQL with Persistent Volume (PV) and Secrets management.
- **CI/CD**: Jenkins Pipeline (`Jenkinsfile`) for automated deployment.
- **Observability**: Prometheus (Data collection) & Grafana (Visualization).

## ⚡ Quick Start (Kind Cluster)

### 1. Create Cluster with Ingress Support
Run this command to create a Kind cluster with ports 80 and 443 exposed:

```bash
cat <<EOF | kind create cluster --config=-
kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4
nodes:
- role: control-plane
  kubeadmConfigPatches: [ "kind: InitConfiguration\nnodeRegistration:\n  kubeletExtraArgs:\n    node-labels: \"ingress-ready=true\"" ]
  extraPortMappings:
  - {containerPort: 80, hostPort: 80, protocol: TCP}
  - {containerPort: 443, hostPort: 443, protocol: TCP}
EOF
```

### 2. Deploy Everything
Apply all manifests in order:

```bash
# Install Ingress Controller
kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/main/deploy/static/provider/kind/deploy.yaml

# Deploy Postgres & Nginx
kubectl apply -f postgresql/
kubectl apply -f nginx/deployment/
kubectl apply -f nginx/service/
kubectl apply -f nginx/ingress/

# Deploy Jenkins & Monitoring
kubectl create namespace monitoring || true
kubectl apply -f monitoring/
kubectl apply -f jenkins/
```

## 🌐 Accessing the Services

### Web Application
Add the following entry to your `/etc/hosts` file:
`127.0.0.1  my-nginx.local`
Then visit: **[http://my-nginx.local](http://my-nginx.local)**

### Monitoring Dashboard (Grafana)
Expose the service and visit **[http://localhost:3000](http://localhost:3000)**:
```bash
kubectl port-forward -n monitoring svc/grafana-service 3000:80
```
*Default Credentials: `admin` / `admin`*

### Jenkins CI/CD
Expose the service and visit **[http://localhost:8080](http://localhost:8080)**:
```bash
kubectl port-forward svc/jenkins-service 8080:8080
```
